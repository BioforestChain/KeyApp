# 00. 文档哲学 (Philosophy)

## 1. AI-First (AI 优先)
白皮书的首要读者是 **AI Agent**，其次才是人类开发者。
*   **显式优于隐式**: 不要让 AI 猜测上下文。必须显式声明依赖关系和代码路径。
*   **结构化优于叙事**: 使用列表、表格和代码块，减少长篇大论的自然语言描述。
*   **单一事实来源**: 代码和文档必须一致。如果文档过时，它就是有害的噪音。

## 2. Deep & Atomic (深度原子化)
*   **深度**: 不要只写"如何做"，要写"为什么这么做" (Design Rationale)。
*   **原子化**: 一个文件只讲一个概念。如果一个文件超过 500 行，它可能需要拆分。

## 3. Code Mapping (代码映射)
文档必须是代码的镜像。
*   **原则**: 看到文档目录就能推导出代码目录，反之亦然。
*   **锚点**: 每个技术文档必须包含 `Code Source` 链接。
