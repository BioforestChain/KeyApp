# 第四篇：服务篇

> 定义产品的「数据怎么流动」

---

## 概述

服务篇回答四个核心问题：

1. **服务如何组织？** → [服务架构](./01-服务架构/)
2. **链交互怎么抽象？** → [链服务接口](./02-链服务接口/)
3. **平台能力怎么封装？** → [平台服务](./03-平台服务/)
4. **数据变化怎么通知？** → [事件系统](./04-事件系统/)

---

## 章节导航

### [第十一章：服务架构](./01-服务架构/)

定义服务分层和 Adapter 模式。

**要点**：
- 服务分层：Chain Adapter + Platform Service
- Adapter 模式：统一接口，多链实现
- Registry：适配器注册与获取

### [第十二章：链服务接口](./02-链服务接口/)

定义与区块链交互的标准接口。

**要点**：
- IAssetService：资产查询
- ITransactionService：交易构建与广播
- IIdentityService：地址派生与签名

### [第十三章：平台服务](./03-平台服务/)

定义平台能力抽象层。

**要点**：
- 生物识别：指纹/面容认证
- 安全存储：加密存储敏感数据
- 剪贴板/Toast/相机等

### [第十四章：事件系统](./04-事件系统/)

定义数据变化的订阅机制。

**要点**：
- Subscribable 接口
- 余额变化、交易状态事件
- 类型安全的事件订阅
