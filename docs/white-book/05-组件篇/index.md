# 第五篇：组件篇

> 定义产品的「UI 组件规范」

---

## 概述

本篇定义 BFM Pay 所需的 UI 组件的**行为规范**，不涉及具体技术实现。任何遵循本规范的实现（React、Vue、Flutter、SwiftUI 等）都应产生一致的用户体验。

---

## 章节导航

### [第十五章：组件规范](./01-组件体系/)

定义核心 UI 组件的行为契约。

**包含**：
- 基础组件规范（按钮、输入框、卡片等）
- 业务组件规范（钱包卡片、地址显示、金额显示等）
- 组件状态机定义

### [第十六章：表单规范](./02-表单系统/)

定义表单的状态管理和验证规范。

**包含**：
- 表单状态机
- 验证时机规范
- 错误反馈规范

### [第十七章：交互规范](./03-组件开发规范/)

定义组件的交互行为和可访问性规范。

**包含**：
- 触摸/点击反馈
- 键盘导航
- 屏幕阅读器支持

---

## 规范符号说明

| 符号 | 含义 |
|-----|------|
| **MUST** | 必须实现 |
| **SHOULD** | 建议实现 |
| **MAY** | 可选实现 |
| **State** | 组件状态 |
| **Event** | 用户/系统事件 |
| **Transition** | 状态转换 |
