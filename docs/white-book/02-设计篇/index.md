# 第二篇：设计篇

> 定义产品的「长什么样」

---

## 概述

设计篇回答三个核心问题：

1. **用户如何操作？** → [交互设计](./01-交互设计/)
2. **界面什么风格？** → [视觉设计](./02-视觉设计/)
3. **设计遵循什么原则？** → [设计原则](./03-设计原则/)

---

## 章节导航

### [第四章：交互设计](./01-交互设计/)

定义信息架构、导航流程和页面结构。

**要点**：
- 底部 Tab 导航：首页、历史、设置
- 栈式导航：原生 App 体验
- 底部弹窗：选择器、确认框

### [第五章：视觉设计](./02-视觉设计/)

定义视觉系统和品牌风格。

**要点**：
- 设计令牌：颜色、字体、间距
- 组件样式：按钮、卡片、表单
- 深色模式支持

### [第六章：设计原则](./03-设计原则/)

定义设计决策的指导原则。

**要点**：
- 信任优先：透明、可控、安全提示
- 渐进复杂度：新手友好、进阶可选
- 响应式设计：适配多种屏幕
